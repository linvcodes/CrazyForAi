A thinking-first, execution-second workflow for building production-ready web apps using AI

Before You Begin: Step 0

Important Context: You are not the developer of this project — you are User 0, developing an app to solve your own problem. You need to give ChatGPT the full context that you are a non-technical user who wants ChatGPT to ask you relevant questions to hone in on a development idea you can execute on Replit.

ChatGPT needs to ensure accuracy by asking you clarifying questions at every step, not by guessing or proposing solutions prematurely.

Step 0 — Confirm Constraints: "Confirm the global constraints listed above with me. Do not proceed until confirmed."
1

Understand the User's Life

ChatGPT asks only these three questions to understand real-life context without jumping to solutions.

Exact prompt from master: Ask me ONLY these three questions: 1. What do you do day-to-day? 2. What part of that feels repetitive, annoying, or mentally draining? 3. What do you constantly re-check, worry about, or feel unsure of? Wait for my answers. Questions ChatGPT will ask:
  1. What do you do day-to-day?
  2. What part of that feels repetitive, annoying, or mentally draining?
  3. What do you constantly re-check, worry about, or feel unsure of?
Output: Raw human context. No solutions, no features.
2

Translate Pain into Problems

Convert answers into 2-3 clear, concrete problems and explain them back in simple, non-technical language.

Exact prompt from master: Translate my answers into 2–3 clear, concrete problems. Explain them back to me in plain language. Ask me to confirm or correct them. Do NOT propose solutions. ChatGPT should:
  • Convert answers into 2-3 clear problems
  • Explain them back in simple language
  • Ask for confirmation or correction
  • NOT propose any solutions
Output: Confirmed problem statements.
3

Validate Demand

Use web search or deep research to verify that other people experience the same problems.

Exact prompt from master: Use web search or deep research to verify that other people experience similar problems. Summarize findings in simple language. Explain why this is a real, recurring problem worth solving. Ask me if I want to proceed. Output: Validated problem worth solving.
4

Lock Domain Language

Explicitly define key terms to prevent semantic ambiguity later.

Exact prompt from master: Explicitly define all key terms relevant to this domain. For each term: - What it means - What it does NOT mean - How it differs from related terms Ask me to confirm the vocabulary. No UI or features yet. ChatGPT defines:
  • Key terms used in the domain
  • What each term means
  • What each term does NOT mean
  • Which concepts must never overlap
Output: Approved domain vocabulary.
5

Define Invariants & Illegal States

Lock the rules that define system truth and must never be violated.

Exact prompt from master: Ask questions to determine: - What values can only increase or decrease - What actions must never be allowed - What states are invalid - What actions change totals vs do not Do not proceed until these rules are confirmed. ChatGPT locks:
  • What values can only increase or decrease
  • What actions must never be allowed
  • What states are invalid
  • What affects totals vs what does not
Output: Confirmed invariant rules.
6

Define State Mutation Rules

Explain in plain language what happens internally when each action occurs. No UI. No code.

Exact prompt from master: Ask questions to define: - What happens internally when each action occurs - How values change (direction and magnitude) - When changes apply (instant, scheduled, monthly) - Whether actions are reversible (undo) Summarize the full logic model and ask for confirmation. ChatGPT explains:
  • What happens internally when each action occurs
  • How values change
  • When changes apply (instant, monthly, scheduled)
  • Whether actions are reversible
Output: Approved logic model.
7

Map Intent to UI

Map one user action to one intent. No overloaded controls. No mixed intent.

Exact prompt from master: Ask questions to map: - One user action → one intent - Which actions are drag-based - Which actions are button-based - Which actions must be separated to avoid confusion Produce a UI behavior summary and ask for approval. ChatGPT maps:
  • One user action → one intent
  • Buttons vs drag actions
  • Separate sections for separate concepts
  • Clear separation between different types of actions
Output: Final UI behavior specification.
8

Logic Confirmation Gate

Before any code is generated, summarize the entire system and ask for explicit approval.

Exact prompt from master: Summarize the entire system: - Problem - Rules - Logic - UI behavior Ask for explicit approval to proceed. If rejected, return to the relevant step. ChatGPT must summarize:
  • Problem
  • Rules
  • Logic
  • UI behavior

If rejected, return to the relevant step.

Output: Final confirmation to build.
9

Generate Final Build Prompt

ChatGPT produces ONE complete build prompt that encodes everything. This is the distilled result of the entire chat conversation.

Exact prompt from master: Generate ONE complete, Replit-ready build prompt that: - Encodes all clarified user needs - Encodes all rules, invariants, and illegal states - Encodes all UI behavior - Encodes all persistence requirements - Explicitly states what must NOT be included - Assumes Replit is only an executor, not a decision-maker The prompt must: - Contain no placeholders - Contain no future work - Contain no explanations after it OUTPUT ONLY THE FINAL BUILD PROMPT. The build prompt must:
  • Encode all clarified needs
  • Encode all rules and constraints
  • Encode all UI behavior
  • Encode all data persistence requirements
  • Explicitly state what must NOT be included

No placeholders. No future work. No explanations after the prompt.

Output: Single Replit-ready build prompt.

ChatGPT Master Prompt

Copy this prompt to ChatGPT to start the structured conversation

You are my AI product guide and specification engine.

Your job is to guide me through a structured conversation that results in ONE final, production-ready build prompt I can paste directly into Replit to build a working web app.

IMPORTANT RULES:
- Assume I am non-technical.
- You must ask questions ONE STEP AT A TIME.
- Do NOT jump ahead.
- Do NOT propose features or solutions until explicitly allowed.
- Do NOT generate any code until the very end.
- Do NOT allow yourself to guess behavior.
- All ambiguity must be resolved through questions.
- Everything you reason about must be distilled into the final build prompt.
- The final build prompt must leave NOTHING unspecified.

GLOBAL CONSTRAINTS (ALWAYS ENFORCE):
- Single-page application
- Offline-capable
- No APIs, no keys, no authentication
- File-based data persistence (local storage or JSON files)
- Minimal user input → automatic processing → clear output
- Beginner-friendly UX
- One final build prompt only

PROCESS (YOU MUST FOLLOW THIS ORDER EXACTLY):

────────────────────────
Confirm the global constraints listed above with me.
Do not proceed until confirmed.

2. What part of that feels repetitive, annoying, or mentally draining?

Wait for my answers.

- Explicitly states what must NOT be included

The prompt must:
- Contain no placeholders
- Contain no future work
- Contain no explanations after it

OUTPUT ONLY THE FINAL BUILD PROMPT.
